# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Monorepo of reusable Terraform modules organized by cloud provider and service category. Modules are consumed via Git source references and versioned independently with git tags.

## Repository Layout

- `modules/<provider>/<category>/<module>/` — module source (e.g., `modules/aws/storage/dynamo-db/`)
- `examples/<provider>/<category>/<module>/` — usage examples mirroring the module path
- `tests/` — test directory (currently empty)
- `tag-module.sh` — version tagging script

## Common Commands

```sh
# Format all Terraform files
terraform fmt -recursive

# Validate a specific module
terraform -chdir=modules/aws/storage/dynamo-db init -backend=false
terraform -chdir=modules/aws/storage/dynamo-db validate

# Run pre-commit hooks (format, validate, docs, lint)
pre-commit run --all-files

# Tag a module release (default is patch bump)
./tag-module.sh <module-name>              # patch
./tag-module.sh <module-name> --minor      # minor
./tag-module.sh <module-name> --major      # major
```

## Pre-commit Hooks

Hooks run automatically on commit via `pre-commit-terraform`:
- `terraform_fmt` — formatting
- `terraform_validate` — validation
- `terraform_docs` — auto-generates README.md for modules under `modules/`
- `terraform_tflint` — linting

Plus standard hooks: trailing-whitespace, end-of-file-fixer, check-merge-conflict, detect-private-key.

## CI (PR Validation)

GitHub Actions on PRs to `main` runs `terraform fmt -check -recursive` and `terraform init/validate` on every directory under `modules/` and `examples/`.

## Module Conventions

- Each module must have `main.tf`, `variables.tf`, and `outputs.tf`
- README.md files inside modules are auto-generated by `terraform-docs` — do not edit them manually
- Every module needs a corresponding example under `examples/` at the matching path
- Modules use `terraform >= 1.0` version constraint
- Tags follow the format `<module-name>/v<major>.<minor>.<patch>` (e.g., `dynamo-db/v1.0.0`)

## PR Checklist

PRs must pass: `terraform fmt`, `terraform validate`, and have README updated via terraform-docs (handled by pre-commit).
